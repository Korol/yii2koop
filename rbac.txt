Пользователи -> пользователи сайта, прошедшие регистрацию.

Маршруты -> указать, какие маршруты (routes) будем использовать для разрешений.

Разрешения -> создаём нужные нам разрешения для Админов, Контент-менеджеров,
    Зарегистрированных пользователей(если для них будут доступны некоторые страницы сайта,
    которые недоступны простым посетителям сайта).
    Указываем доступные Маршруты для каждого разрешения (Админ, КМ, Юзер, Гость).

Роли -> это и есть собственно роли пользователей в общем понимании. Каждой роли присваиваются определенные Разрешения.

Назначение -> это связь Пользователей с Ролями.

Принцип и порядок работы с RBAC:
1. (Маршруты) Выбираем Маршруты, которые будут использоваться в RBAC:
    /admin/* - все роуты, которые начинаются с admin
    /rbac/* - все роуты, которые начинаются с rbac
    /private/* - все роуты, которые начинаются с private

2. (Разрешения) Создаём Разрешения для этих Маршрутов, даём им понятные описания:
    adminAccess -> /admin/* - доступ к Админке
    userAccess -> /rbac/* - доступ к RBAC
    privateAccess -> /private/* - доступ к Личному Кабинету на сайте
    маршруты в Разрешениях можно объединять (несколько маршрутов в одном разрешении)

3. (Роли) Создаём Роли пользователей, присваиваем ролям Разрешения, при этом можно и нужно выстраивать
иерархическую структуру – от роли с самым малым доступом к роли с самым высоким доступом:
    user -> privateAccess
    manager -> user + adminAccess (т.е. разрешено всё, что разрешено user-у + доступ к Админке)
    manager -> manager + userAccess (т.е. разрешено всё, что разрешено manager-у + доступ к RBAC)

4. (Назначения) Назначаем пользователям соответствующие их задачам Роли.
